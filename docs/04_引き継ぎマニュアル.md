## 3. 技術アーキテクチャ

### 3.1 設計思想

- **単一HTMLファイル**: サーバー不要で動作。USB配布やローカル保存でもそのまま使える
- **外部依存の最小化**: 外部ライブラリは SheetJS（Excel処理）のCDNのみ
- **クライアント完結**: データ送信なし。プライバシーに配慮した設計

### 3.2 plan.html の内部構造

上から順に以下のセクションで構成されています:

```
<head>
  ├── SheetJS CDN 読み込み
  └── <style> CSS（約350行）

<body>
  ├── #toolbar            ツールバーHTML
  ├── #title-bar          タイトル表示
  ├── #header-info        目標欄（学校目標・学年目標・学級目標・担任情報）
  ├── #table-wrap         テーブル＋SVGコンテナ
  │   ├── #plan-table       計画表テーブル
  │   └── #arrow-svg        矢印描画SVGレイヤー
  ├── #modal-overlay      セル編集モーダル
  ├── #subj-overlay       教科別編集モーダル
  └── <script>
      ├── CURRICULUM_DATA    カリキュラム埋め込みデータ
      ├── 定数・状態管理     MONTHS, ALL_SUBJECTS, state
      ├── initGrade()        学年初期化
      ├── renderTable()      テーブル描画（メイン）
      ├── renderArrows()     矢印SVG描画
      ├── openEditor()       セル編集モーダル
      ├── openSubjectEditor() 教科別編集モーダル
      ├── Header Info        目標欄の編集・保存・読み込み
      ├── Save / Load        JSON保存・読み込み
      ├── Excel Export       Excelテンプレートダウンロード
      ├── Excel Import       Excelインポート
      ├── Extract Mode       抽出版トグル
      └── Init               初期化処理
```

### 3.3 状態管理

```
state = {
  grade: '5',       // 現在の学年（文字列）
  units: {},        // { "教科名\t月番号": [{id, unit, hours}, ...] }
  arrows: []        // [{id, fromId, toId}, ...]
}
```

目標欄のデータはDOMに直接保持され、JSON保存時に `headerInfo` として含まれます。

### 3.4 ビルドプロセス

build.py は curriculum_data.json を読み込み、HTMLテンプレートにJSONデータを埋め込んで plan.html を出力します。

カリキュラムデータを変更した場合は必ず build.py を再実行してから plan.html をコミットしてください。

## 4. 開発フロー

### 4.1 通常の変更手順

1. フィーチャーブランチを作成
2. plan.html または関連ファイルを修正
3. ブラウザで plan.html を直接開いて動作確認
4. プルリクエストを作成
5. main にマージ → GitHub Pages に自動デプロイ

### 4.2 テスト

自動テストはありません。以下を手動で確認してください:

- 各学年（1〜6年）の切り替えが正常に動作する
- セル編集（追加・変更・削除）が正常に保存される
- 矢印の作成・削除が動作する
- 目標欄の入力・保存・読み込みが動作する
- JSON保存 → JSON読み込みでデータが完全に復元される
- Excelテンプレート → Excelインポートでデータが正しく往復する
- PDF出力でA3横長に全体が正しく出力される
- 抽出版モードで自己実現活動関連のみフィルタされる

## 6. 既知の制約

| 項目 | 詳細 |
|------|------|
| 自動テストなし | 全テストが手動 |
| 単一HTMLファイル | ファイルサイズが大きい（約120KB+） |
| Undo/Redo なし | 誤操作の取り消しができない |
| マルチユーザー非対応 | 同時編集・共有機能なし |
| オフラインExcel不可 | Excel機能はCDNのSheetJSに依存 |
| 年度ハードコード | 「令和7年度」がコード内にハードコード |
