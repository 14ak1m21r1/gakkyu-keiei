# 年間単元指導計画表 作成ツール ─ 管理マニュアル

> 本マニュアルは、本ツールの運用管理・カリキュラムデータの更新・トラブル対応を担当する方向けです。

---

## 1. システム構成

### 1.1 アーキテクチャ

- Webアプリ本体: plan.html（単一HTMLファイル）
- カリキュラムデータ: curriculum_data.json
- Excel生成スクリプト: create_plan.py（Python）
- HTMLビルドスクリプト: build.py（Python）
- ホスティング: GitHub Pages（自動デプロイ）

**サーバーレス構成**: 静的HTMLファイルのみ。バックエンドサーバー・データベース不要。

### 1.2 技術スタック

| 要素 | 技術 |
|------|------|
| フロントエンド | HTML5 + CSS3 + Vanilla JavaScript |
| Excel処理（ブラウザ） | SheetJS (xlsx) CDN版 |
| Excel生成（サーバー） | Python + openpyxl |
| HTMLビルド | Python |
| デプロイ | GitHub Actions → GitHub Pages |
| バージョン管理 | Git / GitHub |

### 1.3 外部依存

| 依存先 | 用途 | 障害時の影響 |
|--------|------|-------------|
| SheetJS CDN | Excelインポート/エクスポート | Excel機能が使用不可。他の機能は正常動作 |
| GitHub Pages | ホスティング | アプリにアクセスできない |

---

## 2. デプロイ

### 2.1 自動デプロイ

mainブランチへのプッシュにより GitHub Actions が自動実行され、GitHub Pages に反映されます。

### 2.2 ロールバック

問題が発生した場合：

```
git revert HEAD          # 直前のコミットを取り消す新しいコミットを作成
git push origin main     # デプロイをトリガー
```

## 3. カリキュラムデータの管理

### 3.1 データ構造

curriculum_data.json にすべての学年のカリキュラムデータが格納されています。

```
{
  "1": {
    "国語": [
      {"unit": "春が来た", "month": 4, "hours": 2}
    ]
  }
}
```

| フィールド | 型 | 説明 |
|-----------|-----|------|
| unit | 文字列 | 単元名 |
| month | 数値 | 月（4=4月, ..., 3=3月） |
| hours | 数値 | 配当時数 |

### 3.2 カリキュラムデータの更新手順

1. curriculum_data.json を編集
2. build.py を実行して plan.html にデータを埋め込む
3. 動作確認後、main ブランチにコミット＆プッシュ

> **重要**: plan.html 内にはカリキュラムデータが埋め込まれています。curriculum_data.json を編集しただけでは反映されません。必ず build.py を実行してください。

### 3.3 年度の更新

新年度のタイトル（「令和○年度」）は plan.html 内のタイトルバー生成部分で設定されています。年度が変わる際にはこの値を更新し、ビルド・デプロイしてください。

## 4. トラブルシューティング

| 問題 | 原因 | 対処 |
|------|------|------|
| アプリが表示されない | GitHub Pages のデプロイ失敗 | Actions タブでエラーを確認 |
| Excel機能が動かない | SheetJS CDN にアクセスできない | ネットワーク環境を確認 |
| カリキュラムデータが古い | build.py 未実行 | build.py を実行してビルド |
| 印刷で色が出ない | ブラウザの印刷設定 | 「背景のグラフィック」にチェック |
| 保存したJSONが読めない | ファイル破損 or 形式不正 | grade と units フィールドが必須 |

### デバッグ

- ブラウザの開発者ツール（F12）のコンソールでエラーを確認できます
- データの状態はコンソールで `state` と入力すると確認できます

---

## 5. セキュリティ

- **認証なし**: 誰でもアクセス可能なツールです（公開前提）
- **データ送信なし**: すべての処理がブラウザ内で完結
- **個人情報**: 児童名等を含むデータをツール内に入力しないでください
- **ファイル管理**: 保存されたJSON/Excelファイルは各自で適切に管理してください
